# ME Constitution

ME is a local meaning environment operated through Codex App.

Prompts guide the model. Transactions govern the product.

## Rule 1: Thought Is Not Cognition

A Thought is something the user gives ME.

A Cognition is a Thought the user explicitly chose to keep.

## Rule 2: Capture Is Not Approval

Casual add, save, remember, capture, note, or put-in-ME wording captures
a Thought only.

It does not approve the Thought as a Cognition.

## Rule 3: Output Is Not Cognition

Codex Output remains Output.

To keep something from Output, the user must bring it back as a Thought.

## Rule 4: Reference Is Not Cognition

A Reference is material Codex may consult.

A Reference does not become a Cognition by being imported, read, or
cited.

## Rule 5: Procedure Is Not Cognition

A Procedure tells Codex how to work.

A Procedure does not become a Cognition.

## Rule 6: Transaction Beats Prompt

If a skill and the engine disagree, the engine wins.

Invalid mutation must fail even if Codex asks for it.

## Rule 7: Read Does Not Mutate

Search, context, comparison, drafting, and inspection must not advance
the current Snapshot.

## Rule 8: Render The State

Every ordinary user response must show whether ME changed or did not
change.
